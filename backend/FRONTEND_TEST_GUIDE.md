# 前端测试页面使用指南

## 📖 概述

`test_frontend.html` 是一个独立的 HTML 测试页面，可以直接在浏览器中打开，用于测试后端 API 的所有功能。

## 🚀 快速开始

### 步骤 1：确保后端服务正在运行

```bash
cd backend
venv\Scripts\python.exe run_dev.py
```

确认看到以下输出：
```
INFO:     Uvicorn running on http://0.0.0.0:8000
```

### 步骤 2：打开测试页面

**方式一：直接双击文件**
- 在文件资源管理器中找到 `backend/test_frontend.html`
- 双击即可在默认浏览器中打开

**方式二：拖拽到浏览器**
- 将 `test_frontend.html` 文件拖拽到浏览器窗口

**方式三：右键打开**
- 右键点击文件 → 选择"打开方式" → 选择浏览器（Chrome、Edge、Firefox 等）

## 🔧 常见问题

### Q1: 点击按钮没有任何反应？

**可能原因**：
1. 后端服务未启动
2. 浏览器控制台有错误

**解决方法**：
1. 确认后端服务正在运行（http://localhost:8000/docs 可访问）
2. 按 F12 打开浏览器开发者工具，查看 Console 标签页的错误信息

---

### Q2: 提示 "Failed to fetch" 或网络错误？

**可能原因**：
1. CORS 跨域问题
2. 后端服务地址不正确

**解决方法**：
1. 检查后端配置中的 `ALLOWED_ORIGINS` 是否包含您的页面地址
2. 如果使用本地文件打开（file://），需要在后端添加 `"*"` 到允许的来源

修改 `backend/app/core/config.py`：
```python
ALLOWED_ORIGINS: str = "*"  # 开发环境允许所有来源
```

---

### Q3: 登录后刷新页面，还是显示"未登录"？

**原因**：Token 应该保存在 localStorage，刷新后会自动恢复

**检查方法**：
1. 按 F12 打开开发者工具
2. 切换到 Application 标签
3. 左侧选择 Local Storage
4. 查看是否有 `authToken` 键

如果没有，可能是浏览器禁用了 localStorage

---

### Q4: 注册时提示邮箱已存在？

**原因**：该邮箱已经被注册过

**解决方法**：
1. 使用不同的邮箱地址
2. 或者直接登录（如果记得密码）
3. 或删除数据库重新测试：删除 `backend/wordstyle.db` 文件

3. 点击"获取个人信息" → 应该成功
4. 清除 Token：在浏览器控制台执行 `localStorage.removeItem('authToken')`
5. 刷新页面 → 显示"未登录"
6. 重新登录

1. 不登录，直接点击"创建订单" → 应该被禁用
2. 不登录，直接点击"开始转换" → 应该被禁用
3. 登录后，这两个按钮应该启用

## 🛠️ 高级用法

### 修改 API 地址

如果您的后端运行在其他端口，编辑 `test_frontend.html` 第 329 行：

```javascript
const BASE_URL = 'http://localhost:8000';  // 修改为您的地址
```

### 查看网络请求

1. 按 F12 打开开发者工具
2. 切换到 Network 标签
3. 点击任意按钮
4. 查看发送的请求和响应

可以查看：
- 请求 URL
- 请求方法（GET/POST）
- 请求头（包括 Authorization）
- 请求体
- 响应状态码
- 响应数据
