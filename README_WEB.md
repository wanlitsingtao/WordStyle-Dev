# 📄 文档转换工具 - Web 版 (MVP)

基于 Streamlit 的快速原型版本，用于验证付费模式的可行性。

---

## 🚀 快速开始

### 1. 安装依赖

```bash
pip install -r requirements_web.txt
```

### 2. 启动应用

**Windows:**
```bash
双击 "启动Web版.bat"
```

**Mac/Linux:**
```bash
streamlit run app.py
```

### 3. 访问应用

浏览器会自动打开：http://localhost:8501

## 📖 使用流程

1. **上传源文档** - 选择需要转换的 Word 文档 (.docx)
2. **上传模板** - 选择定义了目标样式的模板文档
3. **配置选项**（可选）：
   - 语气转换
   - 插入应答句
   - 自定义列表符号
4. **点击"开始转换"**
5. **下载结果** - 下载转换后的文档

## 🔧 技术架构

### 核心技术栈
- **前端**: Streamlit (Python Web 框架)
- **后端**: doc_converter.py (现有转换引擎)
- **存储**: 本地 JSON 文件 (用户数据)
- **部署**: Streamlit Cloud / Render (免费托管)

### 目录结构
```
LingMa/
├── app.py                    # Streamlit 主应用
├── doc_converter.py          # 核心转换引擎
├── requirements_web.txt      # Web 版依赖
├── 启动Web版.bat             # Windows 启动脚本
├── user_data.json            # 用户数据（自动生成）
└── temp_*.docx              # 临时文件（自动清理）
```

## 🌐 部署到公网

### 方案 A: Streamlit Cloud（推荐，最简单）

1. 将代码推送到 GitHub
2. 访问 https://streamlit.io/cloud
3. 连接 GitHub 仓库
4. 一键部署

**优点**: 
- ✅ 完全免费
- ✅ 自动 HTTPS
- ✅ 无需配置服务器

**缺点**:
- ⚠️ 冷启动较慢（约30秒）
- ⚠️ 每月750小时免费额度

### 方案 B: Render

1. 创建 `render.yaml` 配置文件
2. 推送到 GitHub
3. 在 Render 上连接仓库

**优点**:
- ✅ 免费套餐充足
- ✅ 性能较好

### 方案 C: 自建服务器

使用 VPS + Docker 部署，适合长期运营。

## 📊 数据说明

### 用户数据存储
- **位置**: `user_data.json`（本地文件）
- **内容**: 
  ```json
  {
    "user_xxx": {
      "credits": 3,
      "last_reset": "2026-04",
      "total_converted": 5
  ```

## 🐛 常见问题

### Q: 如何修改管理员联系方式？
A: 编辑 `app.py` 第 18 行：
```python
ADMIN_CONTACT = "微信号：your_wechat_id"  # 改成你的微信号
```

### Q: 如何调整免费额度？
A: 编辑 `app.py` 第 15-16 行：
```python
FREE_CREDITS_PER_MONTH = 3  # 每月免费次数
MAX_PAGES_FREE = 50         # 最大页数
```

### Q: 转换失败怎么办？
A: 检查以下几点：
1. 文档格式是否为 .docx
2. 模板文档是否正确
3. 查看错误日志

### Q: 如何手动给用户增加额度？
A: 编辑 `user_data.json`，找到对应用户，修改 `credits` 值。
