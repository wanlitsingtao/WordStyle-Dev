#### 3.2.3 system_config 表（系统配置）

**数据库表结构**：
```sql
CREATE TABLE system_config (
    id SERIAL PRIMARY KEY,
    config_key VARCHAR(100) UNIQUE NOT NULL,
    config_value TEXT NOT NULL,
    description VARCHAR(500),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

**字段说明**：

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `id` | SERIAL | 配置项ID | - |
| `config_key` | VARCHAR(100) | 配置键（唯一） | - |
| `config_value` | TEXT | 配置值 | - |
| `description` | VARCHAR(500) | 配置描述 | NULL |
| `updated_at` | TIMESTAMPTZ | 最后更新时间 | NOW() |

**示例配置**：
```json
{
  "free_paragraphs_daily": "10000",
  "paragraph_price": "0.001",
  "max_file_size_mb": "50",
  "admin_contact": "微信号：your_wechat_id"
}
```

   def load_config_from_db(key: str, default_value=None):
       """从数据库加载配置"""
       if DATA_SOURCE == "api":
           response = requests.get(f"{BACKEND_URL}/api/config/{key}")

## 💡 注意事项

### 1. 配置缓存策略

**问题**：频繁读取数据库会影响性能

**解决方案**：
```python
# 使用Redis或内存缓存
from functools import lru_cache
import time

CONFIG_CACHE = {}
CACHE_TTL = 300  # 5分钟

def get_config_with_cache(key: str):
    """带缓存的配置读取"""
    now = time.time()
    
    # 检查缓存是否有效
    if key in CONFIG_CACHE:
        value, timestamp = CONFIG_CACHE[key]
        if now - timestamp < CACHE_TTL:
            return value
    
    # 从数据库读取
    value = load_config_from_db(key)
    
    # 更新缓存
    CONFIG_CACHE[key] = (value, now)
    
    return value
```

### 2. 配置验证

**问题**：非法配置可能导致系统异常

**解决方案**：
```python
def validate_config(key: str, value: str) -> bool:
    """验证配置值的合法性"""
    validators = {
        'paragraph_price': lambda v: float(v) > 0,
        'free_paragraphs_daily': lambda v: int(v) >= 0,
        'admin_contact': lambda v: len(v) > 0 and len(v) <= 500,
    }
    
    validator = validators.get(key)
    if validator:
        try:
            return validator(value)
        except:
            return False
    return True
```

### 3. 配置回滚

**问题**：错误配置可能导致系统不可用

**解决方案**：
```sql
-- 创建配置历史表
CREATE TABLE config_history (
    id SERIAL PRIMARY KEY,
    config_key VARCHAR(100),
    old_value TEXT,
    new_value TEXT,
    changed_by VARCHAR(50),
    changed_at TIMESTAMPTZ DEFAULT NOW()
);

-- 更新配置时记录历史
INSERT INTO config_history (config_key, old_value, new_value, changed_by)
SELECT config_key, config_value, NEW.config_value, current_user
FROM system_config WHERE config_key = NEW.config_key;
```

### 4. 多实例同步

**问题**：多个服务实例的配置不一致

**解决方案**：
- 使用Redis Pub/Sub通知配置变更
- 或者设置较短的缓存TTL（如5分钟）
- 或者提供手动刷新配置的按钮
