**修改位置**: 第43-63行

```python
# ✅ 设置每日定时清理任务（每天零点执行）
try:
    from apscheduler.schedulers.background import BackgroundScheduler
    from file_manager import schedule_daily_cleanup
    
    scheduler = BackgroundScheduler()
    # 每天零点执行清理
    scheduler.add_job(
        func=schedule_daily_cleanup,
        trigger='cron',
        hour=0,
        minute=0,
        id='daily_file_cleanup',
        name='每日文件清理任务'
    )
    scheduler.start()
    logger.info("✅ 每日文件清理任务已启动（每天零点执行）")
except ImportError:
    logger.warning("⚠️ APScheduler未安装，跳过定时清理任务")
except Exception as e:
    logger.warning(f"定时清理任务启动失败: {e}")
```

## 📊 实现效果预期

### 自动清理流程

```
服务启动
  ↓
执行cleanup_on_startup()
  ↓
清理所有临时文件（temp_source_*, temp_template_*）
  ↓
清理超过7天的转换结果文件
  ↓
启动APScheduler定时任务（每天零点执行）
  ↓
每次转换完成后
  ↓
清理本次转换产生的临时文件
```

### 文件保留策略

| 文件类型 | 保留时间 | 清理时机 |
|---------|---------|---------|
| temp_source_* | 不保留 | 转换完成后立即清理 |
| temp_template_* | 不保留 | 转换完成后立即清理 |
| result_*.docx | 7天 | 服务启动时 + 每天零点 |

## 📝 注意事项

1. **APScheduler兼容性**
   - Streamlit可能不支持后台线程
   - 如果定时任务不工作，考虑使用外部cron job

2. **文件锁问题**
   - 删除正在下载的文件可能导致错误
   - 建议在删除前检查文件是否被占用

3. **权限问题**
   - 确保应用有读写conversion_results目录的权限
   - Render等云平台可能需要特殊配置

4. **性能考虑**
   - 文件数量多时分页很重要
   - 避免一次性加载所有文件信息
