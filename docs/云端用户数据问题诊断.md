# WordStyle 云端用户数据问题诊断指南

## 问题描述
- 前端（Streamlit Cloud）已生成用户ID
- 管理后台显示"没有用户数据"
- Render 后端可能未重新部署

## 诊断步骤

### 1. 检查后端服务是否正常运行

访问以下 URL 确认后端服务状态：

```
https://wordstyle-backend.onrender.com/api/health
```

**预期响应**：
```json
{"status": "healthy", "timestamp": "2026-05-13T..."}
```

如果无法访问或返回错误，说明后端服务未运行。

---

### 2. 检查用户注册 API 是否存在

访问以下 URL 测试用户注册 API：

```bash
# 使用 curl 测试
curl -X POST https://wordstyle-backend.onrender.com/api/admin/users/test_user_123 \
  -H "Content-Type: application/json" \
  -d '{"balance": 0, "paragraphs_remaining": 10000, "total_paragraphs_used": 0, "total_converted": 0}'
```

**预期响应**：
```json
{
  "success": true,
  "user_id": "test_user_123",
  "message": "用户数据已保存"
}
```

如果返回 404，说明 API 端点不存在（后端代码未部署）。

---

### 3. 检查数据库中是否有用户数据

登录 Supabase Dashboard，执行以下 SQL：

```sql
-- 查看所有用户
SELECT id, balance, paragraphs_remaining, total_converted, created_at, last_login
FROM users
ORDER BY created_at DESC
LIMIT 10;

-- 统计用户总数
SELECT COUNT(*) as total_users FROM users;
```

如果没有用户记录，说明用户注册 API 调用失败。

---

### 4. 检查前端是否正确调用 API

在 Streamlit Cloud 的控制台中查看日志，应该能看到类似这样的输出：

```
🌐 数据源模式: API (后端: https://wordstyle-backend.onrender.com)
✅ 数据访问层初始化：API 模式
新用户 xxxxxxxxxxxx 已创建并领取 10000 免费段落
```

如果没有这些日志，说明前端代码可能有问题。

---


1. **修改 Streamlit Cloud Secrets**：

2. **重新部署前端**：

3. **注意**：

## 常见问题

### Q1: Render 服务休眠怎么办？

Render 免费套餐在 15 分钟无请求后会进入休眠状态。

**解决方案**：
- 定期访问后端健康检查端点保持活跃
- 或使用 UptimeRobot 等监控服务定期 ping

### Q2: 如何确认用户数据已写入数据库？

在 Supabase SQL Editor 中执行：

```sql
SELECT * FROM users WHERE id = '您的用户ID';
```

替换 `'您的用户ID'` 为实际的用户ID（可在前端侧边栏看到）。

### Q3: API 调用失败如何调试？

在前端添加调试日志：

```python
# data_manager.py 第 504 行附近
except Exception as e:
    print(f"⚠️ API 请求失败: {method.upper()} {endpoint} - {e}")
    import traceback
    traceback.print_exc()  # 打印完整堆栈
    return {}
```

然后在 Streamlit Cloud 日志中查看详细错误信息。

## 快速验证脚本

创建一个简单的测试脚本验证端到端流程：

```python
import requests

BACKEND_URL = "https://wordstyle-backend.onrender.com"

# 1. 测试健康检查
print("1. 测试健康检查...")
response = requests.get(f"{BACKEND_URL}/api/health")
print(f"   状态码: {response.status_code}")
print(f"   响应: {response.json()}")

# 2. 测试用户注册
print("\n2. 测试用户注册...")
test_user_id = "test_api_user_001"
user_data = {
    "balance": 0.0,
    "paragraphs_remaining": 10000,
    "total_paragraphs_used": 0,
    "total_converted": 0
}
response = requests.post(
    f"{BACKEND_URL}/api/admin/users/{test_user_id}",
    json=user_data
)
print(f"   状态码: {response.status_code}")
print(f"   响应: {response.json()}")

# 3. 测试获取用户列表
print("\n3. 测试获取用户列表...")
response = requests.get(f"{BACKEND_URL}/api/admin/users?limit=10")
print(f"   状态码: {response.status_code}")
result = response.json()
print(f"   用户总数: {result.get('total', 0)}")
print(f"   用户列表: {len(result.get('users', []))} 个")

# 4. 测试任务统计
print("\n4. 测试任务统计...")
response = requests.get(f"{BACKEND_URL}/api/admin/task-stats")
print(f"   状态码: {response.status_code}")
print(f"   响应: {response.json()}")
```

保存为 `test_backend_api.py` 并运行：
```bash
python test_backend_api.py
```

## 总结

**最可能的原因**：
1. ✅ 后端服务未运行或已休眠
2. ✅ 后端代码未部署（缺少用户注册 API）
3. ✅ 数据库连接配置错误
4. ❌ 前端代码问题（可能性较低）

**建议操作顺序**：
1. 先检查后端服务是否可访问
2. 再检查 API 端点是否存在
3. 最后检查数据库中是否有数据
4. 如有必要，重新部署后端服务
