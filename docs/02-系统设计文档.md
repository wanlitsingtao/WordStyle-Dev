## 1. 系统架构概述

### 1.1 整体架构

WordStyle 采用**前后端分离架构**，通过 **API 模式**实现数据访问。

```
┌─────────────────────────┐         HTTP API         ┌─────────────────────────┐
│   前端 (Streamlit Cloud) │ ──────────────────────> │   后端 (Render)          │
│                          │                         │                          │
│  • 用户界面              │ <────────────────────── │  • FastAPI 服务          │
│  • 文档上传              │     JSON 响应           │  • 数据库操作            │
│  • 转换配置              │                         │  • 业务逻辑              │
│  • 结果下载              │                         │                          │
└─────────────────────────┘                         └──────────┬──────────────┘
                                                               │
                                                               │ PostgreSQL 连接
                                                               ▼
                                                    ┌─────────────────────────┐
                                                    │   数据库 (Supabase)      │
                                                    │                          │
                                                    │  • users 表             │
                                                    │  • orders 表            │
                                                    │  • conversion_tasks 表  │
                                                    │  • system_config 表     │
                                                    └─────────────────────────┘
```

### 1.2 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **前端框架** | Streamlit | Python Web 应用框架 |
| **后端框架** | FastAPI | 高性能 Python API 框架 |
| **数据库** | PostgreSQL (Supabase) | 云端关系型数据库 |
| **ORM** | SQLAlchemy | Python ORM 框架 |
| **数据验证** | Pydantic | 数据模型和验证 |

| **文档处理** | python-docx | Word 文档处理库 |
| **依赖管理** | pip + requirements.txt | Python 包管理 |

#### 2.1.2 数据访问层设计

**统一数据访问接口**（`data_manager.py`）：

系统支持三种数据源模式，通过环境变量自动切换：

```python
# 数据源检测逻辑
if BACKEND_URL and USE_SUPABASE:
    DATA_SOURCE = 'api'  # API 模式
elif DATABASE_URL and USE_SUPABASE:
    DATA_SOURCE = 'supabase'  # Supabase 直连模式
else:
    DATA_SOURCE = 'local'  # 本地模式（SQLite + JSON）
```

**三种模式对比**：

| 特性 | API 模式 | Supabase 模式 | Local 模式 |
|------|---------|--------------|-----------|
| **使用场景** | Streamlit Cloud | Render 等允许出站连接的环境 | 本地开发 |
| **数据访问** | HTTP API | SQLAlchemy ORM | SQLite + JSON |
| **性能** | 中等（网络延迟） | 高（直连数据库） | 最高（本地文件） |
| **可靠性** | 依赖后端服务 | 依赖数据库连接 | 文件系统稳定性 |
| **扩展性** | 最佳（前后端分离） | 良好 | 有限 |

**当前生产环境**：API 模式

#### 2.1.3 用户识别与持久化

**核心机制**：
- **基于客户端设备标识生成唯一用户ID（12位MD5哈希字符串）**
- **用户初次访问时自动在数据库中创建用户记录**
- **所有用户数据持久化到数据库（users表）**
- **无需注册登录，降低使用门槛**

**用户ID生成流程**：
```python
# 1. 获取客户端设备标识
client_ip = headers.get('X-Forwarded-For', '127.0.0.1')
user_agent = headers.get('User-Agent', 'unknown')
device_key = f"{client_ip}|{user_agent}"

# 2. 生成设备指纹
device_fingerprint = md5(device_key)[:16]

# 3. 生成用户ID
user_id = md5(f"wordstyle_device_{device_fingerprint}")[:12]

# 4. 查询数据库
user = db.query(User).filter(User.id == user_id).first()

if user:
    # 用户已存在，更新最后登录时间
    user.last_login = datetime.now()
    db.commit()
else:
    # 新用户，创建记录并赠送免费额度
    user = User(
        id=user_id,
        paragraphs_remaining=10000,  # 免费额度
        is_active=True,
        created_at=datetime.now(),
        last_login=datetime.now()
    )
    db.add(user)
    db.commit()
```

**数据持久化内容**：
- ✅ 用户基本信息（ID、用户名、创建时间）
- ✅ 额度信息（余额、剩余段落数、累计使用）
- ✅ 转换历史（每次转换的记录）
- ✅ 活跃信息（最后登录时间、累计转换次数）
- ✅ 样式映射配置（用户自定义的样式映射）

### 2.3 文档转换引擎

#### 2.3.1 核心技术

**库**：python-docx

**转换流程**：
```
1. 读取源文档
   ↓
2. 分析源文档样式
   ↓
3. 读取模板文档
   ↓
4. 分析模板文档样式
   ↓
5. 生成样式映射关系
   ↓
6. 逐段落转换
   ├─ 复制文本内容
   ├─ 应用目标样式
   ├─ 处理表格
   ├─ 处理图片
   └─ 处理特殊元素（Visio、合并单元格等）
   ↓
7. 插入应答句（如配置）
   ↓
8. 语气转换（如配置）
   ↓
9. 保存输出文档
```

**表格处理**：
- 支持合并单元格

- 处理复杂嵌套表格

**图片处理**：
- 保持图片尺寸比例
- 支持多种图片格式（PNG、JPG、GIF）
- 处理嵌入对象（Visio）

**特殊元素处理**：

### 3.2 表结构设计

#### 3.2.1 users 表（用户信息）

```sql
CREATE TABLE users (
    id VARCHAR(12) PRIMARY KEY,          -- 12位字符串ID（应用层生成）
    username VARCHAR(50),                -- 用户名
    style_mappings JSONB DEFAULT '{}'::jsonb, -- 样式映射配置
    balance FLOAT DEFAULT 0.0,           -- 账户余额（元）
    paragraphs_remaining INTEGER DEFAULT 0,  -- 剩余段落数
    total_paragraphs_used INTEGER DEFAULT 0, -- 累计使用段落数
    total_converted INTEGER DEFAULT 0,   -- 累计转换文件数
    is_active BOOLEAN DEFAULT TRUE,      -- 是否激活
    created_at TIMESTAMPTZ DEFAULT NOW(), -- 创建时间
    last_login TIMESTAMPTZ,              -- 最后登录时间
    updated_at TIMESTAMPTZ DEFAULT NOW()  -- 更新时间
);
```

**字段说明**：

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `id` | VARCHAR(12) | 用户唯一标识，12位 MD5 哈希字符串 | - |
| `username` | VARCHAR(50) | 用户名 | NULL |
| `style_mappings` | JSONB | 样式映射配置（JSON格式） | `'{}'::jsonb` |
| `balance` | FLOAT | 账户余额（元） | 0.0 |
| `paragraphs_remaining` | INTEGER | 剩余可用段落数 | 0 |
| `total_paragraphs_used` | INTEGER | 累计已用段落数 | 0 |
| `total_converted` | INTEGER | 累计转换文件数 | 0 |
| `is_active` | BOOLEAN | 用户是否激活 | TRUE |
| `created_at` | TIMESTAMPTZ | 用户创建时间 | NOW() |
| `last_login` | TIMESTAMPTZ | 最后登录时间 | NULL |
| `updated_at` | TIMESTAMPTZ | 最后更新时间 | NOW() |

**索引**：
- 主键索引：`users_pkey` on `id`

**注意事项**：
- ✅ **不包含** `wechat_openid`、`email`、`password_hash` 等未使用的字段
- ✅ **不包含** `balance` 以外的计费相关字段（简化设计）
- ✅ `id` 字段类型为 `VARCHAR(12)`，与应用程序生成的用户ID一致
- ✅ `style_mappings` 字段用于存储用户自定义的样式映射配置
- ✅ 总字段数：11个字段

#### 3.2.2 conversion_tasks 表（转换任务）

```sql
CREATE TABLE conversion_tasks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id VARCHAR(12) NOT NULL,          -- 用户ID（外键）
    source_file VARCHAR(500),              -- 源文件路径
    template_file VARCHAR(500),            -- 模板文件路径
    converted_file VARCHAR(500),           -- 转换后文件路径
    status VARCHAR(20) DEFAULT 'pending',  -- 任务状态
    progress INTEGER DEFAULT 0,            -- 进度（0-100）
    error_message TEXT,                    -- 错误信息
    completed_at TIMESTAMPTZ,              -- 完成时间
    created_at TIMESTAMPTZ DEFAULT NOW(),  -- 创建时间
    updated_at TIMESTAMPTZ DEFAULT NOW(),  -- 更新时间
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

**字段说明**：

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `id` | UUID | 任务唯一标识 | gen_random_uuid() |
| `user_id` | VARCHAR(12) | 关联用户ID（外键） | - |
| `source_file` | VARCHAR(500) | 源文件路径 | NULL |
| `template_file` | VARCHAR(500) | 模板文件路径 | NULL |
| `converted_file` | VARCHAR(500) | 转换后文件路径 | NULL |
| `status` | VARCHAR(20) | 任务状态（pending、processing、completed、failed） | 'pending' |
| `progress` | INTEGER | 转换进度（0-100） | 0 |
| `error_message` | TEXT | 错误信息（失败时） | NULL |
| `completed_at` | TIMESTAMPTZ | 任务完成时间 | NULL |
| `created_at` | TIMESTAMPTZ | 任务创建时间 | NOW() |
| `updated_at` | TIMESTAMPTZ | 最后更新时间 | NOW() |

**索引**：
- 主键索引：`conversion_tasks_pkey` on `id`
- 外键索引：`conversion_tasks_user_id_fkey` on `user_id`

**注意事项**：
-  不包含 `task_id`、`filename`、`paragraphs`、`cost`、`output_files` 字段
- ✅ 使用 `source_file`、`template_file`、`converted_file` 存储文件路径
- ✅ 使用 `updated_at` 跟踪任务更新时间

#### 4.3.3 Render部署问题排查

**常见问题及解决方案**：

1. **Alembic迁移失败 - ModuleNotFoundError: No module named 'app'**
   - **原因**：Python找不到app模块,因为alembic/env.py未正确设置sys.path
   - **解决**：在`backend/alembic/env.py`文件开头添加:
     ```python
     import sys
     import os
     sys.path.insert(0, os.path.dirname(os.path.dirname(os.path.abspath(__file__))))
     ```

2. **数据库连接失败 - could not translate host name**
   - **原因**：使用了错误的连接池器地址格式或用户名格式
   - **解决**：
     - 使用Supabase提供的标准连接池器地址:`aws-{region}.pooler.supabase.com:6543`
     - 用户名格式必须为:`postgres.{project_id}`
     - 示例:`postgresql://postgres.cgfdhubkklpyvjgezeeq:密码@aws-1-ap-south-1.pooler.supabase.com:6543/postgres`

3. **URL编码错误 - invalid interpolation syntax**
   - **原因**：DATABASE_URL中的`%`(如`%40`)被Alembic的configparser误解析为插值语法
   - **解决**：在`backend/alembic/env.py`中转义百分号:
     ```python
     escaped_url = database_url.replace('%', '%%')
     config.set_main_option("sqlalchemy.url", escaped_url)
     ```

4. **IPv6连接失败 - Network is unreachable**
   - **原因**：Render尝试通过IPv6连接Supabase直连地址失败
   - **解决**：改用连接池器地址(端口6543),而非直连地址(端口5432)

5. **启动命令未执行Alembic迁移**
   - **原因**：Render缓存了旧的render.yaml配置
   - **解决**：在Render控制台手动修改Start Command为:`alembic upgrade head && uvicorn app.main:app --host 0.0.0.0 --port $PORT`

### 4.4 数据流

#### 4.4.1 用户首次访问流程

```
1. 用户打开 Streamlit Cloud 应用
   ↓
2. app.py 初始化
   - 检查 session_state 中是否有 user_id
   - 如无，生成新的 12 位 user_id（基于 hostname 哈希）
   ↓
3. 调用 data_manager.load_user_data(user_id)
   - 在 API 模式下，调用后端 API: GET /api/users/{user_id}
   ↓
4. 调用 claim_free_paragraphs(user_id)
   - 检查是否是今日首次访问
   - 如是，赠送 10,000 免费段落
   ↓
5. 调用 register_or_login_user(user_id, user_data)
   - 在 API 模式下，调用后端 API: POST /api/users/{user_id}
   - 后端执行 SQL: INSERT INTO users (...)
   ↓
6. 用户数据写入 Supabase 数据库
   - id: 12位字符串
   - balance: 0.0
   - paragraphs_remaining: 10000
   - total_converted: 0
   - is_active: True
   - created_at: NOW()
   - last_login: NOW()
```

#### 4.4.2 文档转换流程

```
1. 用户上传文档并点击"开始转换"
   ↓
2. 前端调用后端 API 创建任务
   - POST /api/tasks
   - 请求体: {user_id, filename, paragraphs, cost}
   ↓
3. 后端创建任务记录
   - INSERT INTO conversion_tasks (...)
   - 返回 task_id
   ↓
4. 前端轮询任务状态
   - GET /api/tasks/{task_id}
   - 每 2 秒查询一次
   ↓
5. 后端执行转换（异步）
   - 更新任务状态: PENDING → PROCESSING → COMPLETED
   - 更新进度: 0 → 100
   ↓
6. 转换完成
   - 更新用户数据: paragraphs_remaining -= paragraphs
   - 更新用户统计: total_converted += 1
   ↓
7. 前端显示下载链接
```
