## 可能的原因

### 原因1：云端部署未完成 ⏳

**现象**：
- Streamlit Cloud/Render还在拉取代码或构建中
- 部署队列较长，需要等待

**解决方法**：
1. 登录Streamlit Cloud控制台
2. 查看应用部署状态
3. 等待部署完成（通常5-10分钟）

### 原因2：云端缓存问题 🔄

**现象**：
- 云端仍在使用旧版本的代码
- Python模块缓存未清除

**解决方法**：
1. 在Streamlit Cloud控制台点击"Redeploy"按钮
2. 或者提交一个空commit触发重新部署：
   ```bash
   git commit --allow-empty -m "触发重新部署"
   git push origin main
   ```

### 原因3：依赖安装问题 📦

**现象**：
- 云端环境没有正确安装依赖
- requirements.txt缺少必要的包

**检查清单**：
- [ ] requests库是否已安装
- [ ] supabase库是否已安装（如果使用Supabase模式）
- [ ] Python版本是否兼容

**解决方法**：
1. 检查`requirements.txt`是否包含：
   ```
   requests>=2.28.0
   supabase>=2.0.0  # 如果使用Supabase模式
   ```
2. 在云端控制台查看构建日志
3. 确认依赖安装成功

### 原因4：数据源模式配置错误 ⚙️

**现象**：
- 云端使用的DATA_SOURCE模式不正确
- BACKEND_URL或DATABASE_URL环境变量未设置

**检查方法**：
在云端应用的日志中查找：
```
[INFO] 数据源模式: API (后端: https://...)
或
[INFO] 数据源模式: Supabase (直接连接)
或
[INFO] 数据源模式: 本地 (SQLite + JSON)
```

**解决方法**：
1. 检查Streamlit Cloud的Secrets配置
2. 确保设置了正确的环境变量：
   ```toml
   # .streamlit/secrets.toml
   USE_SUPABASE = true
   BACKEND_URL = "https://your-backend.onrender.com"
   
   [supabase]
   url = "https://xxx.supabase.co"
   key = "your_key"
   ```

### 原因5：Python路径问题 📁

**现象**：
- 云端的工作目录结构与本地不同
- 模块导入路径不正确

**检查方法**：
在admin_web.py中添加调试信息：
```python
import sys
st.write(f"Python路径: {sys.path}")
st.write(f"当前目录: {os.getcwd()}")
```

## 推荐解决步骤

### 步骤1：检查云端部署状态

1. 登录Streamlit Cloud控制台
2. 找到你的应用
3. 查看"Deployments"标签页
4. 确认最新部署的状态是"Running"而不是"Building"或"Failed"

### 步骤2：查看部署日志

1. 在Streamlit Cloud控制台点击"Logs"
2. 查找启动时的日志输出
3. 确认看到以下信息：
   ```
   [OK] 已加载 .env 配置文件
   🌐 数据源模式: API (后端: https://...)
   ```

### 步骤3：手动触发重新部署

如果部署状态正常但仍报错：

1. 在Streamlit Cloud控制台点击"Redeploy"
2. 等待新的部署完成
3. 刷新页面测试

### 步骤4：提交空commit触发部署

如果控制台没有Redeploy按钮：

```bash

```
