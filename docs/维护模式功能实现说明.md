```python
# ✅ 检查维护模式（在所有业务逻辑之前）
try:
    from data_manager import get_config
    maintenance_mode = get_config('maintenance_mode')
    if maintenance_mode and maintenance_mode.lower() == 'true':
        # 如果启用维护模式，显示维护提示并停止执行
        st.markdown("""
        <div style="text-align: center; padding: 50px;">
            <h1>🔧 系统维护中</h1>
            <p>系统正在进行升级维护，请稍后再试。</p>
        </div>
        """, unsafe_allow_html=True)
        st.stop()  # 停止执行后续代码
except Exception as e:
    logger.warning(f"维护模式检查失败（不影响服务）: {e}")
```

## 🎯 使用方法

### 启用维护模式

1. 访问管理后台: `http://localhost:8502`
2. 进入「系统配置」页面
3. 展开「🔧 维护模式配置」
4. 勾选「启用维护模式」
5. 点击「💾 保存维护模式」
6. 系统提示"✅ 维护模式已启用 - 用户将被重定向到维护页面"

### 验证效果

1. 访问用户前端: `http://localhost:8501`
2. 应该看到维护提示页面
3. 直接访问维护页面: `streamlit run maintenance_page.py`

### 禁用维护模式

1. 访问管理后台: `http://localhost:8502`
2. 进入「系统配置」页面
3. 展开「🔧 维护模式配置」
4. 取消勾选「启用维护模式」
5. 点击「💾 保存维护模式」
6. 系统提示"✅ 维护模式已禁用 - 系统恢复正常"

## 🔍 技术细节

### 数据流向

```
用户访问 app.py
    ↓
检查 maintenance_mode 配置
    ↓
如果为 'true':
    → 显示维护提示
    → st.stop() 停止执行
如果为 'false':
    → 继续正常业务流程
```

### 访问控制

| 访问路径 | 维护模式启用时 | 维护模式禁用时 |
|---------|--------------|--------------|
| app.py (用户前端) | 显示维护提示 | 正常访问 |
| admin_web.py (管理后台) | 正常访问 | 正常访问 |
| maintenance_page.py (维护页面) | 显示维护页面 | 显示维护页面 |

### 配置持久化

- 配置存储在Supabase PostgreSQL数据库的`system_config`表中
- 支持三种数据源模式：API、Supabase直连、Local
- 配置变更立即生效，无需重启服务

## ⚠️ 注意事项

1. **启用前检查**：确保管理后台可以正常访问
2. **维护时机**：建议在低峰期进行维护
3. **及时关闭**：维护完成后及时关闭维护模式
4. **管理员权限**：只有管理员可以访问管理后台修改配置
5. **用户体验**：维护页面应提供清晰的提示和预计恢复时间
