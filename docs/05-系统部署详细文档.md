## 3. Supabase数据库配置

### 3.1 创建Supabase项目

1. 登录 [Supabase](https://supabase.com)
2. 点击 "New Project"
3. 填写项目信息:
   - Name: wordstyle
   - Database Password: 设置强密码
   - Region: 选择靠近用户的区域(如Singapore)
4. 等待项目创建完成(约2分钟)

### 3.2 获取数据库连接信息

1. 进入项目 → Settings → Database
2. 找到 "Connection string" 部分
3. 复制以下两种连接字符串:

**直连地址(Session mode)**:
```
postgresql://postgres:[YOUR-PASSWORD]@db.{project_id}.supabase.co:5432/postgres
```

**连接池器地址(Transaction mode)**:
```
postgresql://postgres.{project_id}:[YOUR-PASSWORD]@aws-{region}.pooler.supabase.com:6543/postgres
```

**重要**: 
- **Render部署必须使用连接池器地址**(端口6543)
- 用户名格式为 `postgres.{project_id}` (带项目ID)
- 主机名格式为 `aws-{region}.pooler.supabase.com`

### 3.3 执行数据库迁移

#### 方法1: 通过Render自动执行(推荐)

Render启动时会自动执行`alembic upgrade head`,无需手动操作。

#### 方法2: 本地手动执行

```bash
cd backend

# 设置环境变量
export DATABASE_URL="postgresql://postgres.{project_id}:密码@aws-{region}.pooler.supabase.com:6543/postgres"

# 执行迁移
alembic upgrade head
```

### 3.4 验证数据库表

连接数据库后,确认以下表已创建:
- `users`: 用户信息表
- `conversion_tasks`: 转换任务表
- `system_config`: 系统配置表
- `comments`: 评论表
- `feedbacks`: 反馈表
- `style_mappings`: 样式映射表

## 5. Streamlit Cloud前端部署

### 5.1 创建Streamlit App

1. 登录 [Streamlit Cloud](https://streamlit.io/cloud)
2. 点击 "New app"
3. 配置应用:
   - **Repository**: wanlitsingtao/WordStyle
   - **Branch**: main
   - **Main file path**: app.py
   - **App name**: wordstyle

### 5.2 配置Secrets

在Streamlit Cloud控制台 → Settings → Secrets 中添加:

```toml
# Streamlit Cloud 配置（API模式）
# 注意：同时配置 BACKEND_URL 和 USE_SUPABASE=true 时，会自动检测为 API 模式
BACKEND_URL = "https://wordstyle-backend.onrender.com"
USE_SUPABASE = true  # 启用数据库功能（非本地SQLite）

# DATABASE_URL 在 API 模式下不需要，但保留用于切换模式
DATABASE_URL = "postgresql://postgres.{project_id}:密码@aws-{region}.pooler.supabase.com:6543/postgres"
```

**数据源模式自动检测逻辑**：
- **API模式**：BACKEND_URL存在 + USE_SUPABASE=true → 通过后端API访问数据库（Streamlit Cloud）
- **Supabase直连模式**：DATABASE_URL存在 + USE_SUPABASE=true + 无BACKEND_URL → 直接连接数据库（Render后端）
- **Local模式**：USE_SUPABASE=false 或无数据库配置 → 使用本地SQLite + JSON文件（本地开发）

**注意**: 
- BACKEND_URL必须指向Render部署的后端API
- USE_SUPABASE设置为true启用数据库功能
- Streamlit Cloud会自动检测为API模式

## 9. 故障排查指南

### 9.1 Render部署失败

#### 问题1: Build失败

**症状**: 部署日志显示pip install错误

**原因**: 
- requirements.txt缺失或格式错误
- 依赖包版本冲突

**解决**:
```bash
# 检查requirements.txt
cat backend/requirements.txt

# 重新生成依赖文件
pip freeze > backend/requirements.txt
```

#### 问题2: Start Command未执行Alembic迁移

**症状**: 日志显示直接启动Uvicorn,没有Alembic输出

**原因**: Render缓存了旧的render.yaml配置

**解决**:
1. 在Render控制台 → Settings
2. 找到 "Start Command"
3. 手动修改为: `alembic upgrade head && uvicorn app.main:app --host 0.0.0.0 --port $PORT`
4. 保存并触发重新部署

#### 问题3: ModuleNotFoundError: No module named 'app'

**症状**: Alembic迁移失败,提示找不到app模块

**原因**: alembic/env.py未正确设置sys.path

**解决**: 确认`backend/alembic/env.py`开头包含:
```python
import sys
import os
sys.path.insert(0, os.path.dirname(os.path.dirname(os.path.abspath(__file__))))
```

### 9.2 数据库连接失败

#### 问题1: could not translate host name

**症状**: DNS解析失败

**原因**: 
- 使用了错误的连接池器地址
- 用户名格式错误

**解决**:
- 确认使用Supabase提供的标准连接池器地址
- 用户名格式: `postgres.{project_id}`
- 主机名: `aws-{region}.pooler.supabase.com`
- 端口: `6543`

#### 问题2: Network is unreachable

**症状**: IPv6连接失败

**原因**: 尝试通过IPv6连接Supabase直连地址

**解决**: 改用连接池器地址(端口6543)

#### 问题3: invalid interpolation syntax

**症状**: Alembic解析DATABASE_URL失败

**原因**: URL中的`%`被configparser误解析

**解决**: 确认`backend/alembic/env.py`中转义百分号:
```python
escaped_url = database_url.replace('%', '%%')
config.set_main_option("sqlalchemy.url", escaped_url)
```

### 9.3 前端无法连接后端

#### 问题1: CORS错误

**症状**: 浏览器控制台显示CORS policy错误

**原因**: ALLOWED_ORIGINS配置错误

**解决**: 
- 确认Render环境变量ALLOWED_ORIGINS设置为`https://*.streamlit.app`
- 重启Render服务

#### 问题2: API返回404

**症状**: 前端调用API返回404

**原因**: BACKEND_URL配置错误

**解决**:
- 检查Streamlit Secrets中BACKEND_URL是否正确
- 确认后端服务正常运行: `curl https://wordstyle-backend.onrender.com/health`

### 9.4 用户数据无法写入

#### 问题1: 用户ID类型不匹配

**症状**: SQL错误,提示类型不匹配

**原因**: users表的id字段类型不是VARCHAR(12)

**解决**:
```sql
-- 检查表结构
\d users

-- 如果类型错误,执行迁移修正
alembic upgrade head
```

#### 问题2: 外键约束失败

**症状**: 插入数据时外键约束错误

**原因**: 引用了不存在的用户ID

**解决**: 确认先创建用户,再创建关联记录

## 11. 常见问题FAQ

### Q1: 为什么使用连接池器而不是直连?

**A**: Render的网络环境无法通过IPv6连接Supabase直连地址,而连接池器使用IPv4,兼容性更好。此外,连接池器可以更好地管理数据库连接,提高性能。

### Q2: 免费套餐的限制是什么?

**A**: 
- **Render**: 每月750小时免费时长,服务休眠后首次访问需等待30秒唤醒
- **Streamlit Cloud**: 完全免费,无限制
- **Supabase**: 500MB数据库存储,每月2GB带宽

### Q3: 如何升级到付费套餐?

**A**: 
- **Render**: 控制台 → Settings → Upgrade Plan
- **Supabase**: 控制台 → Settings → Billing → Upgrade

### Q4: 服务休眠后会影响用户体验吗?

**A**: Render免费套餐的服务在15分钟无访问后会休眠,下次访问需等待30秒唤醒。建议:
- 使用uptime监控服务保持活跃
- 或升级到付费套餐禁用休眠

### Q5: 如何回滚到之前的版本?

**A**: 
1. 在GitHub上revert commit
2. 推送后Render自动重新部署
3. 或在Render控制台手动选择之前的部署版本
