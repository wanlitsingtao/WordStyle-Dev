# 数据源模式切换指南

## 📋 三种模式说明

本项目支持三种数据源模式，可以根据需要灵活切换：

### 1️⃣ **Local模式**（本地开发）
- **数据存储**：本地SQLite数据库 + JSON文件
- **适用场景**：离线开发、功能测试、无需云端同步
- **优点**：无需网络、速度快、完全独立
- **缺点**：数据不共享、无法多设备同步

### 2️⃣ **Supabase模式**（直连数据库）
- **数据存储**：云端Supabase PostgreSQL数据库
- **适用场景**：本地开发但需要云端数据同步、Render免费额度用尽时
- **优点**：实时同步、多设备共享、节省Render部署次数
- **缺点**：需要网络连接、暴露数据库连接信息

### 3️⃣ **API模式**（通过后端API）
- **数据存储**：云端Supabase PostgreSQL数据库（通过Render后端访问）
- **适用场景**：生产环境、Streamlit Cloud部署、安全性要求高
- **优点**：安全性高、业务逻辑集中、适合云端部署
- **缺点**：依赖Render服务、消耗Render免费额度

## 🔧 模式切换方法

### 切换到 Local模式

**步骤**：
1. 编辑 `.env` 文件
2. 设置 `USE_SUPABASE=false`
3. 重启应用

```env
# .env 文件
USE_SUPABASE=false
# BACKEND_URL=... （可选，不影响）
# DATABASE_URL=... （可选，不影响）
```

**启动命令**：
```bash
streamlit run app.py --server.port 8501      # 用户前端
streamlit run admin_web.py --server.port 8502 # 管理后台
```

**验证**：启动日志应显示 `[INFO] 数据源模式: 本地 (SQLite + JSON)`

---

### 切换到 Supabase模式（直连）

**步骤**：
1. 编辑 `.env` 文件
2. 设置 `USE_SUPABASE=true`
3. **注释掉** `BACKEND_URL`
4. 确保 `DATABASE_URL` 已配置
5. 重启应用

```env
# .env 文件
USE_SUPABASE=true

# Supabase 数据库连接字符串
DATABASE_URL=postgresql://postgres.{project_id}:{password}@aws-{region}.pooler.supabase.com:6543/postgres

# 注释掉 BACKEND_URL
# BACKEND_URL=https://wstest-backend.onrender.com
```

**启动命令**：
```bash
streamlit run app.py --server.port 8501      # 用户前端
streamlit run admin_web.py --server.port 8502 # 管理后台
```

**验证**：启动日志应显示 `数据源模式: Supabase (直接连接)`

**注意**：
- 系统会自动将连接池器URL（端口6543）转换为直连URL（端口5432）
- 不需要设置 `SUPABASE_KEY`（使用SQLAlchemy直接连接）

---

### 切换到 API模式

**步骤**：
1. 编辑 `.env` 文件
2. 设置 `USE_SUPABASE=true`
3. **取消注释** `BACKEND_URL` 并设置正确的地址
4. 确保 `DATABASE_URL` 已配置（后端需要）
5. 重启应用

```env
# .env 文件
USE_SUPABASE=true

# Supabase 数据库连接字符串（后端需要）
DATABASE_URL=postgresql://postgres.{project_id}:{password}@aws-{region}.pooler.supabase.com:6543/postgres

# 后端 API 地址
BACKEND_URL=https://wstest-backend.onrender.com
```

**启动命令**：
```bash
# 先启动后端服务（如果本地测试）
cd backend
uvicorn app.main:app --reload --port 8000

# 然后启动前端
cd ..
streamlit run app.py --server.port 8501      # 用户前端
streamlit run admin_web.py --server.port 8502 # 管理后台
```

**验证**：启动日志应显示 `🌐 数据源模式: API (后端: https://...)`

## 🚀 快速切换脚本

### Windows批处理脚本

**切换到Local模式** (`switch_to_local.bat`)：
```batch
@echo off
echo 切换到 Local 模式...
powershell -Command "(Get-Content .env) -replace 'USE_SUPABASE=true', 'USE_SUPABASE=false' | Set-Content .env"
echo ✅ 已切换到 Local 模式
echo 💡 请重启应用以生效
pause
```

**切换到Supabase模式** (`switch_to_supabase.bat`)：
```batch
@echo off
echo 切换到 Supabase 直连模式...
powershell -Command "(Get-Content .env) -replace 'USE_SUPABASE=false', 'USE_SUPABASE=true' | Set-Content .env"
powershell -Command "(Get-Content .env) -replace '^BACKEND_URL=', '# BACKEND_URL=' | Set-Content .env"
echo ✅ 已切换到 Supabase 模式
echo 💡 请重启应用以生效
pause
```

**切换到API模式** (`switch_to_api.bat`)：
```batch
@echo off
echo 切换到 API 模式...
powershell -Command "(Get-Content .env) -replace 'USE_SUPABASE=false', 'USE_SUPABASE=true' | Set-Content .env"
powershell -Command "(Get-Content .env) -replace '^# BACKEND_URL=', 'BACKEND_URL=' | Set-Content .env"
echo ✅ 已切换到 API 模式
echo 💡 请重启应用以生效
pause
```

## ⚠️ 注意事项

### 1. Render免费额度限制
- Render免费账户每月有750小时的运行时间
- 每次Git推送都会触发重新部署（耗时2-3分钟）
- **建议**：本地调试时使用Supabase直连模式，避免频繁部署

### 2. 数据一致性
- 三种模式访问的是**同一个Supabase数据库**（Local模式除外）
- Local模式使用独立的SQLite数据库，数据不同步
- 切换模式前请确认数据来源

### 3. 环境变量优先级
```
Streamlit Secrets > .env文件 > 系统环境变量 > 默认值
```

### 4. 常见问题

**Q: 为什么Supabase模式显示"Invalid URL"？**
A: 检查DATABASE_URL格式是否正确，确保包含完整的项目ID和密码

**Q: Local模式如何迁移数据到云端？**
A: 目前不支持自动迁移，需要手动导出导入

**Q: API模式和Supabase模式有什么区别？**
A: API模式通过后端API访问数据库，更安全；Supabase模式直接连接数据库，速度更快
