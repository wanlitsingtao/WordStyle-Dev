# 安全漏洞修复：防止用户伪造身份领取免费额度

## 🚨 漏洞描述

**严重级别**：高危（High）

**漏洞类型**：身份伪造 / 业务逻辑绕过

**影响范围**：云端部署环境（https://wordstyle.streamlit.app）

### 漏洞原理

在修复前，程序从URL参数中读取用户ID：
```python
# ❌ 旧代码（存在漏洞）
if 'uid' in st.query_params:
    url_user_id = st.query_params['uid']
    st.session_state.user_id = url_user_id  # 直接信任URL参数
```

**攻击方式**：
1. 用户访问 `https://wordstyle.streamlit.app/?uid=222356562671`
2. 修改URL中的uid为任意值：`?uid=999999999999`
3. 系统识别为新用户，分配10000免费段落额度
4. 用户可以无限次修改uid，领取无限免费额度

### 实际影响

- **经济损失**：用户可以无限获取免费转换额度
- **资源滥用**：服务器资源被恶意消耗
- **数据混乱**：数据库中产生大量虚假用户记录

---

## ✅ 修复方案

### 核心思路

**不再信任URL参数中的用户ID**，改为基于设备指纹识别用户身份。

### 修复内容

#### 1. 移除URL参数恢复逻辑

```python
# ❌ 删除的代码
url_user_id = None
if hasattr(st, 'query_params'):
    params = st.query_params
    if 'uid' in params:
        url_user_id = params['uid']

if url_user_id and len(url_user_id) == 12:
    st.session_state.user_id = url_user_id
```

#### 2. 使用设备指纹识别

```python
# ✅ 新代码（安全）
# 获取客户端设备指纹（IP + User-Agent）
headers = st.context.headers if hasattr(st, 'context') and hasattr(st.context, 'headers') else {}
client_ip = headers.get('X-Forwarded-For', '').split(',')[0].strip()
if not client_ip:
    client_ip = headers.get('X-Real-IP', '')
if not client_ip:
    client_ip = '127.0.0.1'

user_agent = headers.get('User-Agent', 'unknown')
device_key = f"{client_ip}|{user_agent}"
device_fingerprint = hashlib.md5(device_key.encode()).hexdigest()[:16]

# 从本地映射文件查找该设备的用户ID
user_mapping_file = Path(__file__).parent / "user_mapping.json"
if user_mapping_file.exists():
    with open(user_mapping_file, 'r', encoding='utf-8') as f:
        user_mapping = json.load(f)
        if device_fingerprint in user_mapping:
            existing_user_id = user_mapping[device_fingerprint]
            st.session_state.user_id = existing_user_id
```

#### 3. 删除URL参数写入逻辑

```python
# ❌ 删除的代码
st.query_params['uid'] = new_user_id  # 避免在URL中暴露用户ID
```

## 🔒 安全性分析

### 修复后的防护机制

| 攻击方式 | 修复前 | 修复后 |
|---------|--------|--------|
| 修改URL参数uid | ✅ 成功 | ❌ 无效 |
| 清除浏览器缓存 | ✅ 可能成功 | ❌ 无效（基于IP+UA） |
| 更换浏览器 | ✅ 成功 | ⚠️ 可能成功（UA不同） |
| 使用代理/VPN | ✅ 成功 | ⚠️ 可能成功（IP不同） |

### 剩余风险与缓解措施

**风险1**：用户更换IP或User-Agent可以生成新用户ID

**缓解措施**：
- 对于普通用户，更换IP/UA的成本较高
- 可以在后端添加速率限制和异常检测
- 监控同一IP段的频繁注册行为

**风险2**：本地环境无法持久化user_mapping.json（云端只读文件系统）

**当前状态**：
- 本地开发：✅ 正常工作（可读写文件）
- Streamlit Cloud：⚠️ 每次重启会丢失映射（需要改进）

**建议改进**：
- 将user_mapping存储到数据库（Supabase PostgreSQL）
- 或使用Redis缓存服务
- 或在会话状态中持久化（session_state）
